# Branch Workflow

## 当前分支

- 当前迭代：`V3`
- 官方分支：`main`
- 当前稳定分支：`hermesv0.9.0`
- 当前开发分支：`012-workspace`

### 升级整合分支

- 只在吸收官方新版本时临时创建。
- 例如下一代整合分支可以命名为：`013`
- 以最新 `main` 为基础创建。
- 用来整合当前稳定分支中的私有改动。

## 日常开发流程

`hermesv0.9.0 -> 012-workspace -> test -> hermesv0.9.0`

## 官方升级流程

`main -> 013 <- hermesv0.9.0`

1. 更新 `main` 到官方最新。
2. 从 `main` 新建升级整合分支。
3. 把当前稳定分支中仍然需要保留的私有改动带到整合分支。
4. 在整合分支上完成冲突处理和回归测试。
5. 测试通过后，整合分支接替为新的当前稳定分支。

## 硬规则

1. 不在 `main` 上写私有功能。
2. 不把 `main` 直接 merge 到旧开发分支。
3. 不把新的整合分支再回灌到旧开发分支。
4. 新一代稳定分支接班后，旧稳定分支只保留为历史参考。

## 一句话原则

平时只维护一条当前稳定分支和一条当前开发分支；升级时临时开整合分支，测通后由它接班。
