# Branch Workflow V1

## 核心原则

- 我的稳定分支是产品主线。
- 官方 `main` 只是更新来源，不是我的产品主线。
- 官方更新只做节选，不做整线硬合并。

## 当前分支

- 官方来源分支：`main`
- 我的稳定分支：`hermesv0.9.0`
- 我的开发分支：`012-workspace`


## 日常开发流程

`hermesv0.9.0 -> 012-workspace -> test -> hermesv0.9.0`

## 吸收官方更新流程

1. 更新 `main` 到官方最新。
2. 从 `hermesv0.9.0` 新建一个临时整合分支。
3. 从 `main` 中挑选我需要的官方提交，带到这个整合分支。
4. 在这个整合分支上完成适配和测试。
5. 测试通过后，再合回 `hermesv0.9.0`。

`main -> 挑提交 -> 013`

`hermesv0.9.0 -> 013 -> test -> hermesv0.9.0`

## 硬规则

  main -> 挑提交 -> 013
  hermesv0.9.0 -> 013 -> test -> hermesv0.9.0
  hermesv0.9.0 -> new workspace

## 一句话原则

以我的稳定分支为中心开发，官方 `main` 只作为更新素材池，按需节选后再并回我的产品主线。
